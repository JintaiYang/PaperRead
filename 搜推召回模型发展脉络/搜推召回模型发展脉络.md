## TL;DR

召回模型的演进主线是**表示能力的持续提升**与**检索范式的根本变革**。从手工规则（ItemCF/UserCF）到浅层 embedding（Item2Vec/MF），再到深度双塔（DSSM/YouTube DNN）、图表示（PinSage/EGES）、多兴趣表示（MIND/ComiRec），用户和物品的表示越来越丰富。检索范式则从遍历匹配 → ANN 向量检索（Faiss/HNSW）→ 树索引（TDM）→ 路径索引（DR）→ 自回归生成（TIGER/OneRec），从"选择"走向"生成"。2024 年起，生成式召回（HSTU/OneRec）和统一召排架构成为主旋律，级联管线正在被端到端生成式模型逐步替代。

## 一、发展阶段总览

召回模型近十余年的发展可划分为七个阶段，从传统协同过滤到生成式召回，呈现出清晰的演进脉络。以下先给出全景视图，再逐阶段展开。

| 阶段 | 时间 | 核心驱动 | 代表模型 | 关键突破 |
|------|------|----------|----------|----------|
| 传统方法 | ~2015 | 规则 + 协同过滤 | ItemCF, Swing, MF | 用户-物品共现相似度，矩阵分解向量化 |
| Embedding 化召回 | 2013–2017 | NLP 迁移 + DNN | DSSM, Item2Vec, YouTube DNN | 双塔范式确立，DNN 向量化召回 |
| 双塔精细化 | 2018–2020 | 负采样 + 序列增强 | EBR, SDM, Airbnb Emb | 工程化落地，负采样策略，序列双塔 |
| 图召回 | 2018–2021 | GNN + 图结构 | PinSage, EGES, LightGCN | 图结构信息融入 embedding 学习 |
| 树索引与路径召回 | 2018–2021 | 索引结构创新 | TDM, JTM, Deep Retrieval | 突破双塔内积限制，O(log N) 复杂度 |
| 多兴趣召回 | 2019–2022 | 多向量表示 | MIND, ComiRec, SINE | 单用户多向量，捕获多样化兴趣 |
| 生成式召回 | 2023–至今 | 自回归生成 | TIGER, HSTU, OneRec | Semantic ID + 生成检索，统一召排 |

## 二、第一阶段：传统方法（~2015）

这一阶段的召回主要依赖启发式规则和协同过滤。

==ItemCF==（Amazon, 2003）凭借直观的"看了又看"逻辑和低计算成本成为工业界最基础的召回路，至今仍广泛使用。==Swing==（阿里）在 ItemCF 基础上引入"中间用户"的惩罚思想，降低热门 item 对相似度计算的干扰，成为工业界 i2i 召回的主流改进。==UserCF== 从用户维度做相似度匹配，适合社交属性强的场景。

==矩阵分解（MF）==（Netflix Prize, 2006–2009）以 SVD++、ALS 为代表，将用户和物品映射到低维潜因子空间。这可以看作最早的 embedding 召回思想——用户向量和物品向量的内积作为匹配分数，可通过 ANN 检索实现在线召回。Spotify、Netflix 都有大规模部署。

这一阶段的核心矛盾是：规则难以泛化，MF 表达能力有限，且难以融入丰富的 side information。

---

## 三、第二阶段：Embedding 化召回（2013–2017）

这个阶段的核心突破是把 NLP 的 embedding 思想迁移到推荐场景，实现了深度向量化召回。

[[DSSM]]（Microsoft, CIKM 2013）是双塔模型的鼻祖。最初用于搜索场景，query 和 document 各一个 DNN 塔，通过编码后计算余弦相似度。它奠定了"用户塔 + 物品塔 → 内积/余弦匹配"的基本范式，这个范式至今仍是工业召回的主流架构。落地于 Bing 搜索。

[[Item2Vec]]（Microsoft, 2016）将 Word2Vec 的 Skip-gram 思想迁移到推荐场景，把用户行为序列类比为"句子"，物品类比为"单词"，训练物品 embedding 后通过 ANN 做 i2i 召回。方法简单但有效，至今许多公司仍作为基础召回路。

[[DeepWalk]] / [[Node2Vec]]（2014/2016）将图结构引入 embedding 学习。DeepWalk 用随机游走生成序列再用 Word2Vec 训练；Node2Vec 进一步引入 BFS/DFS 参数控制游走偏好，使 embedding 可以在同质性和结构等价性之间权衡。

[[YouTube_DNN]]（Google, RecSys 2016）是划时代的工作，首次系统地将深度学习应用于工业级推荐系统的召回阶段。将召回建模为超大规模多分类问题，用 DNN 学习用户 embedding，并通过最近邻检索实现在线服务。提出的 "example age" 特征、serving 时用最后隐层向量做 ANN 检索等工程方案至今仍被广泛借鉴。落地于 YouTube 推荐，处理数十亿级视频库。

> [!info] 双塔范式的确立
> DSSM (2013) → YouTube DNN (2016) 确立了工业召回的基本范式：离线训练双塔模型学习 user/item embedding → 物品侧预计算 embedding 入库 → 在线用户侧实时计算 embedding → ANN 检索 Top-K。这一架构至今仍是绝大多数推荐系统的主要召回路。

---

## 四、第三阶段：双塔精细化与工程化（2018–2020）

双塔模型的基本架构确立后，研究焦点转向如何把双塔做得更好、更能落地。

[[Airbnb_Embedding]]（Airbnb, KDD 2018）将业务目标融入 embedding 训练：在序列中加入预订 session 的全局上下文、同城 listing 的负采样等。是 embedding 工程化落地的标杆论文，直接影响了 Airbnb 搜索排序的效果。

[[EBR]]（Facebook, KDD 2020）系统性地讨论了双塔召回工程化的关键问题：统一 embedding 框架、hard negative mining 策略、混合负采样（random + hard）、特征工程优化、ANN 检索调优（Faiss）以及全链路优化。是双塔工程化落地最完整的参考论文。落地于 Facebook Search。

[[SDM]]（阿里, CIKM 2019）在双塔框架下引入序列建模（LSTM + Multi-head Attention），用 session 内的短期行为和长期偏好的融合来生成用户 embedding。是"序列增强双塔"方向的代表。

> [!important] 负采样：双塔召回的核心工程问题
> 双塔模型的效果很大程度上取决于负采样策略。从 random negative → in-batch negative → hard negative mining → mixed negative sampling → 对比学习，负采样技术的演进贯穿了整个双塔模型的发展历程。EBR 论文对此有系统性的讨论。

---

## 五、第四阶段：图召回（2018–2021）

图神经网络的兴起为召回带来了新的表示学习范式——利用用户-物品交互图的结构信息增强 embedding。

[[EGES]]（阿里, KDD 2018）全称 Enhanced Graph Embedding with Side Information，在 DeepWalk 基础上融入物品属性（品牌、品类、价格等）作为 side information，通过加权聚合解决冷启动问题。落地于淘宝手猜场景，Embedding 质量显著提升。

[[PinSage]]（Pinterest + Stanford, KDD 2018）是首个工业级图卷积神经网络召回系统。在 GraphSAGE 基础上，通过随机游走采样邻居 + Importance Pooling + Producer-Consumer 小批量训练，解决了 GCN 在十亿级图上的可扩展性问题。处理 30 亿节点、180 亿边的规模。落地于 Pinterest 相关 Pin 推荐。

[[LightGCN]]（SIGIR 2020）简化了图卷积操作，去掉了特征变换和非线性激活，仅保留邻居聚合和层间组合，发现这种极简设计反而效果更好。虽然更偏学术，但其设计思想影响了后续工业图召回的实践。

---

## 六、第五阶段：树索引与路径召回（2018–2021）

这一方向的核心思想是：双塔模型受限于内积形式，无法使用复杂的交互模型做召回。通过构建索引结构，可以用任意复杂模型在 O(log N) 时间内完成全库检索。

[[TDM]]（阿里妈妈, KDD 2018）是树模型召回的开创性工作。用二叉树索引全量物品库，叶节点对应物品，非叶节点表示物品集合。训练深度模型预测每层节点的兴趣概率，通过 beam search 在树上逐层检索，将 O(N) 的召回复杂度降到 O(log N)。后续 [[JTM]]（2019）将树结构学习和模型学习联合优化，[[OTM]]（2019）优化了模型训练目标。落地于阿里妈妈展示广告，是树召回三部曲。

[[Deep_Retrieval]]（Google + 字节, 2020）提出了非 ANN 的新型索引结构：用 K 路 D 层的路径来索引物品，通过 EM 算法学习物品到路径的分配，beam search 在路径上检索。与 TDM 的思路类似但更灵活——一个物品可被多条路径索引，解决了树结构中物品只能归属一个叶节点的局限。

---

## 七、第六阶段：多兴趣召回（2019–2022）

这一方向的核心洞察是：单个 embedding 向量不足以表达用户多样化的兴趣。

[[MIND]]（阿里, CIKM 2019）是多兴趣网络的开创性工作。用胶囊网络（Capsule Network）的动态路由机制从用户行为序列中提取 K 个兴趣向量，每个向量代表一个兴趣方向，分别进行 ANN 召回后合并。落地于天猫首页推荐，显著提升了召回的多样性和准确性。

[[ComiRec]]（阿里 + 清华, KDD 2020）在 MIND 基础上做了两个关键改进：一是用 Self-Attention 替代胶囊路由（ComiRec-SA），降低计算成本；二是提出多兴趣召回结果的可控融合策略，用贪婪算法在相关性和多样性之间做权衡，解决了多路兴趣召回后如何去重合并的工业问题。

[[SINE]]（阿里, WSDM 2021）进一步用稀疏兴趣网络做兴趣提取，通过 concept routing 将行为序列中的信息路由到少量兴趣概念上，减少冗余兴趣表示。

> [!info] 多兴趣召回的演进
> MIND (2019) → ComiRec (2020) → SINE (2021)
> 
> 从胶囊路由 → Self-Attention → 稀疏路由，兴趣提取的效率和质量持续提升，同时召回结果的多样性控制从后处理走向端到端建模。

## 八、第七阶段：生成式召回与统一架构（2023–至今）

这是当前最活跃的前沿方向，核心思想从"检索匹配"转向"生成预测"。

[[TIGER]]（Google, NeurIPS 2023）是生成式召回的里程碑。提出 Semantic ID 概念——用 RQ-VAE 将物品内容特征编码为离散的码字元组（语义 ID），然后训练 Transformer seq-to-seq 模型直接自回归生成下一个物品的语义 ID。不再依赖 ANN 检索，而是端到端地"生成"推荐物品。冷启动效果提升 12%。

[[HSTU]]（Meta, ICML 2024）提出 Hierarchical Sequential Transduction Unit 架构，将推荐重新定义为序列转导任务。万亿参数级别，首次在工业级验证了推荐领域的 Scaling Law。已部署于 Instagram/Facebook，服务数十亿用户。

[[OneRec]]（快手, 2025）是第一个端到端生成式推荐系统在工业界全面超越传统级联架构。用单一 encoder-decoder 模型统一了召回和排序，通过 MoE 架构扩展模型容量，引入 DPO 做偏好对齐。在快手 App 实现规模化部署，用户停留时长提升 1.6%。

[[MTGR]]（美团, 2025）美团的工业级生成式推荐框架，创新地将 HSTU 的序列建模能力和 DLRM 的交叉特征能力融合。CTR +1.31%，单位 orders +1.22%，推理成本反降 12%。已落地美团外卖首页推荐。

> [!warning] Semantic ID：生成式召回的基础设施
> Semantic ID 正在成为几乎所有生成式召回系统的通用基础。Google（TIGER/PLUM）、快手（OneRec）、阿里（TIGER 变体）、Meta（HSTU）都依赖某种形式的 learned item tokenization。具体编码方式各异，但核心思想一致：将物品从连续 embedding 空间映射到离散 token 空间，使得 Transformer 可以自回归地"生成"物品。

## 九、发展趋势总结

当前召回模型的发展呈现五个明显趋势：

**趋势一：从 ANN 检索到自回归生成。** TIGER/OneRec 证明了不需要 ANN 索引也能做召回，生成式检索正在成为新范式。

**趋势二：统一召排管线。** 传统的级联架构（召回 → 粗排 → 精排 → 重排）正在被生成式模型逐步统一。OneRec 和 HSTU 已经展示了单一模型替代整个级联 pipeline 的可能性。

**趋势三：Scaling Law 进入召回领域。** HSTU 首次验证了召回/推荐模型也遵循 Scaling Law，模型参数从百亿到万亿持续带来效果提升。

**趋势四：多模态与 LLM 增强。** 利用 LLM 生成物品的语义描述再编码为 embedding，或直接用多模态编码器融合文本、图像、行为信息做召回，正在成为新趋势。

**趋势五：双塔仍是主力，但在被逐步替代。** 双塔模型凭借简单高效的优势仍是大多数工业系统的主要召回路，但生成式方法正在从"补充路"变为"主力路"。
