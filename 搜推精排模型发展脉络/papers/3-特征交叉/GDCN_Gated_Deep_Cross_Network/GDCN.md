## 一、研究背景与动机

### 1.1 领域现状

DCN V2 将 Cross Network 从秩1升级到全秩后，显著提升了交叉能力。然而，Cross Network 对所有特征维度施加统一的交叉操作，未能区分不同维度的交叉重要性。在实际 CTR 场景中，不同特征维度对预测的贡献差异很大，有些维度的交叉信息有价值，有些则可能引入噪声。

### 1.2 现有方法的局限性

DCN V2 的 Cross Layer 对输入向量的每个维度都等同地进行交叉变换，无法抑制不重要维度的交叉信号。FINAL 引入了门控残差来控制交叉信息的整体比例，但这是层级的全局门控，缺乏维度级别的精细控制。

### 1.3 本文解决方案概述

GDCN 在 DCN V2 的 Cross Layer 中引入了维度级别的门控机制（Gating mechanism），让每个维度的交叉强度可以根据输入内容自适应调节。门控信号来自当前输入，通过 sigmoid 输出 0-1 之间的权重，逐元素地控制交叉信息的通过比例。

## 二、解决方案

### 2.1 核心思想

GDCN 的改进思路非常直觉：在 DCN V2 的交叉操作后，加一个输入自适应的门控来"过滤"交叉结果。不同样本、不同维度的门控值不同，使得模型能选择性地保留有价值的交叉信号、抑制噪声交叉。

### 2.2 整体架构

Gated Cross Layer 的公式为：

$$\mathbf{x}_{l+1} = \mathbf{x}_0 \odot (\mathbf{W}_l \mathbf{x}_l + \mathbf{b}_l) \odot \mathbf{g}_l + \mathbf{x}_l$$

其中 $\mathbf{g}_l = \sigma(\mathbf{W}_g \mathbf{x}_l + \mathbf{b}_g) \in (0,1)^d$ 是维度级别的门控向量。

与 DCN V2 的区别仅在于多了一个 $\odot \mathbf{g}_l$ 操作。当 $\mathbf{g}_l$ 全为 1 时，GDCN 退化为 DCN V2；当某些维度的 $g$ 接近 0 时，对应维度的交叉被抑制。

#### 各模块详细说明

**模块1：Gated Cross Layer**

- **功能**：带门控的显式特征交叉
- **门控计算**：$\mathbf{g}_l = \sigma(\mathbf{W}_g \mathbf{x}_l + \mathbf{b}_g)$
- **额外参数**：每层增加 $O(d^2)$ 或 $O(d)$ 的门控参数（取决于 $\mathbf{W}_g$ 的设计）
- **关键优势**：输入自适应 + 维度级别精细控制

**模块2：与 DNN 的组合**

- 沿用 DCN V2 的 Stacked 或 Parallel 策略

#### 3.3.2 评估指标

- **AUC**、**Logloss**

### 3.3 实验结果与分析

GDCN 在多个数据集上优于 DCN V2，验证了门控机制的有效性。提升幅度在不同数据集上有所差异，特征维度多且噪声较大的场景受益更明显。

### 消融实验

#### 消融结果和分析

- **去掉门控（退化为 DCN V2）**：AUC 下降，验证门控有效
- **全局门控 vs 维度门控**：维度级别门控优于全局单一门控
- **门控网络的复杂度**：简单的单层 FC + sigmoid 就足够，更复杂的门控网络收益微弱

## 四、未来工作建议

### 4.1 作者建议的未来工作

探索门控机制在其他网络结构中的应用，以及更高效的门控设计。

### 4.2 基于分析的未来方向

1. **方向1：稀疏门控**
   - 动机：当前 sigmoid 门控值连续，无法真正"关闭"某些维度
   - 可能的方法：Top-K 或 Straight-Through Estimator 实现硬门控
   - 预期成果：更稀疏的交叉，可能提升泛化

### 4.3 改进建议

1. **改进1：跨层门控共享**
   - 当前问题：每层独立的门控参数
   - 改进方案：共享底层的门控特征提取器
   - 预期效果：减少参数
