## 一、研究背景与动机

### 1.1 领域现状

在电商、外卖、打车等综合性平台上，推荐系统需要同时服务多个业务域。不同域之间存在复杂的相互影响——既有共性（用户的通用偏好），也有特性（域内独特的行为模式）。多域 CTR 预测（Multi-Domain CTR Prediction, MDCTR）是一个重要但困难的问题。

### 1.2 现有方法的局限性

传统多域 CTR 模型（如 STAR、AdaSparse、PEPNet 等）将域编码为离散标识符（域 ID），通过共享底层网络 + 域特定的门控/参数化来区分不同域。这种方式存在三个根本问题：

**语义缺失**：域被编码为离散 ID，丢失了域的丰富语义信息（如"外卖"和"买菜"在语义上有很强的关联性，但作为不同 ID 时模型无法感知）。**跷跷板现象**：多域联合训练时，数据量大的域会主导训练过程，导致其他域性能下降。**泛化性差**：新域上线时缺乏历史数据，无法进行零样本预测。

### 1.3 本文解决方案概述

Uni-CTR 利用 LLM 作为骨干网络，学习层级语义表示来捕获域间共性，同时设计域特定网络捕获各域独有特征。通过掩码损失策略实现域特定网络与 LLM 骨干的解耦，使得新增或移除域时无需修改其他组件。

## 二、解决方案

### 2.1 核心思想

LLM 在预训练阶段已经积累了丰富的世界知识和语义理解能力。不同业务域的特征在 LLM 的语义空间中天然具有可比性和关联性。LLM 的多层结构（从浅层的词汇语义到深层的任务语义）为捕获不同粒度的域间共性提供了天然的层级结构。

### 2.2 整体架构

#### 各模块详细说明

**模块1：特征 Prompt 化**

将每个样本的特征转换为自然语言 Prompt。域标识转换为域名描述（如"Domain: Food Delivery"），用户特征转换为属性描述，Item 特征转换为物品描述。所有描述拼接后作为 LLM 的输入。这种设计使不同域的特征在同一个语义空间中表示，LLM 可以利用预训练知识理解域间的语义关联。

**模块2：LLM 骨干——层级语义表示**

使用 LLM（如 LLaMA）作为骨干网络。LLM 的不同层产生的隐状态被视为不同粒度的语义表示：浅层捕获词汇级和短语级的语义，深层捕获更抽象的任务级语义。关键设计：提取多个中间层的表示，为后续的通用网络和域特定网络提供不同粒度的信息。

![[BPD_50.pdf|800]]

> 图2：层级语义表示的提取和利用。不同域的域特定网络可以从 LLM 的不同层提取信息。

**模块3：通用网络（General Network）**

接收 LLM 最后一层的输出表示，学习所有域之间共享的通用 CTR 预测知识。它是一个轻量级的 MLP 网络。通用网络的核心价值在于零样本预测——当新域上线时，即使没有新域的训练数据，也可以给出合理的预测。

**模块4：域特定网络（Domain-Specific Networks）**

每个域配备一个独立的域特定网络，接收 LLM 不同中间层的表示。不同域可以关注不同粒度的语义信息。最终预测是通用网络和对应域特定网络的输出的融合。

**模块5：掩码损失策略**

训练时，每个样本只计算其所属域的域特定网络的损失（通过掩码屏蔽其他域），而通用网络和 LLM 骨干在所有域的数据上共同训练。这实现了域特定网络与 LLM 骨干的解耦：新增域只需添加新的域特定网络，移除域只需删除对应网络，其他组件不受影响。

**模块6：训练策略**

两阶段训练：第一阶段冻结 LLM 骨干，训练通用网络和域特定网络；第二阶段用 LoRA 微调 LLM 骨干，同时继续训练上层网络。

## 三、实验结果

### 3.1 数据集

| 数据集 | 域数 | 规模 | 用途 |
|--------|------|------|------|
| Amazon Reviews | 多品类 | 大规模 | 多域电商 |
| Douban | 多类型 | 中等 | 多类型内容推荐 |
| AliExpress | 多国家 | 大规模 | 跨国电商 |

### 3.2 实验设置

#### 3.2.1 基线方法

- 单域模型（DeepFM、DCN V2）
- 多域模型（STAR、AdaSparse、PEPNet、MAMDR 等）
- 其他 LLM+CTR 方法

#### 3.2.2 评估指标

- **AUC**（各域）
- **LogLoss**
- **零样本预测性能**

### 3.3 实验结果与分析

Uni-CTR 在三个数据集上的所有域中均显著优于 SOTA 多域 CTR 模型。在域平衡性方面，有效缓解了跷跷板现象——在提升弱势域性能的同时，不会显著损害强势域。

#### 零样本预测

在完全没有目标域训练数据的情况下，仅依靠通用网络即可达到接近有监督训练的性能，远超其他基线模型。证明了 LLM 的语义表示确实捕获了可迁移的跨域知识。

### 消融实验

- LLM 骨干 vs 传统 Embedding：替换 LLM 为传统 Embedding 层后性能显著下降
- 层级表示 vs 单层表示：仅使用最后一层不如层级表示
- 掩码损失 vs 全局损失：掩码损失显著优于全局损失，且使系统获得域灵活扩展能力

## 四、未来工作建议

### 4.1 作者建议的未来工作

论文建议探索更高效的 LLM 推理方案、更多域的扩展验证、以及与在线系统的集成。

### 4.2 基于分析的未来方向

1. **方向1：推理效率优化**
   - 动机：LLM 骨干的推理成本仍然较高
   - 可能的方法：结合 ELEC 的蒸馏思路，将 LLM 知识迁移到轻量模型
   - 预期成果：保持多域建模能力的同时降低推理成本

2. **方向2：动态域发现**
   - 动机：当前域的划分是预定义的
   - 可能的方法：让模型自动发现数据中的"域"结构
   - 预期成果：更灵活的多域建模

### 4.3 改进建议

1. **改进1：域特定网络的自适应层选择**
   - 当前问题：每个域从哪一层提取信息是预设的
   - 改进方案：引入可学习的层选择机制
   - 预期效果：自动发现每个域最适合的语义粒度

#### 5.2.1 值得关注的技术点

- LLM 层级语义表示对应多粒度域共性——不同域可以关注不同层的语义
- 掩码损失策略实现的"即插即用"域管理——对工业系统非常友好
- 零样本跨域预测能力——新域上线无需冷启动
